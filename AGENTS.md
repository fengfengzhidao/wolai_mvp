# AGENTS.md

## 项目背景

项目名称：`wolai_mvp`

这是一个参考 wolai / Notion 交互形态的个人笔记 MVP，但当前目标不是复刻成熟产品，而是围绕第一版最小闭环持续演进。

第一版最小闭环：

- 用户进入应用
- 创建一个页面
- 输入标题和正文
- 保存内容
- 回到页面列表
- 再次打开刚才的页面
- 能继续编辑

当前产品定位：

- 单用户个人笔记工作台
- 左侧页面树，右侧块编辑区
- 数据暂存在浏览器 `localStorage`
- 优先验证创建、编辑、保存、再次打开，而不是完整知识库能力

## 技术栈

- Vue 3
- Vite
- 原生 CSS
- `localStorage` 持久化

开发命令：

```bash
npm install
npm run dev
```

构建验证：

```bash
npm run build
```

## 当前代码结构

- `src/App.vue`：组合左侧页面列表和右侧编辑器
- `src/components/Sidebar.vue`：页面列表、新建页面、右键菜单
- `src/components/PageTreeNode.vue`：左侧页面树节点递归渲染
- `src/components/NoteEditor.vue`：页面标题、块编辑器和页面更新逻辑
- `src/components/BlockEditor.vue`：块级输入、键盘交互、斜杠菜单、块右键菜单和多选
- `src/components/CodeBlock.vue`：CodeMirror 6 代码块编辑、高亮、语言选择和复制
- `src/composables/useNotes.js`：页面数据、当前页面、创建、选择、删除、保存
- `src/utils/blockOperations.js`：块创建、转换、复制、删除、移动等通用操作
- `src/utils/codeLanguages.js`：代码块语言列表和 CodeMirror 语言扩展映射
- `src/utils/formatTime.js`：时间格式化
- `src/styles/base.css`：全局布局和编辑器样式

## 当前数据结构

页面结构：

```js
{
  id: string,
  title: string,
  content: string,
  blocks: Block[],
  parentId: string | null,
  order: number,
  createdAt: number,
  updatedAt: number
}
```

块结构：

```js
{
  id: string,
  type: "paragraph" | "heading1" | "heading2" | "bullet" | "numbered" | "todo" | "code",
  text: string,
  checked: boolean,
  language?: string
}
```

说明：

- `blocks` 是当前正文编辑的主数据。
- `content` 目前保留作兼容字段，由 blocks 文本拼接生成。
- `parentId` / `order` 用于左侧页面树；页面仍以平铺数组保存在 `localStorage`，渲染时组装树。
- 旧页面数据读取时会自动补齐 `blocks`、`parentId`、`order` 等字段。
- 不要在当前阶段引入数据库、后端或账号系统。

## 重要历史决策

- 曾接入过 `@toast-ui/editor` 做 Markdown 实时编辑，后来按用户要求回退。
- 当前方向改为自建最小块级编辑体验，而不是 Markdown 编辑器。
- 阶段 3 曾暂时跳过，后来已补齐右键块菜单、拖拽移动、复制、删除和框选批量删除能力。
- 代码块选择稳健路线：使用 CodeMirror 6 负责编辑态和实时语法高亮，而不是手写代码编辑器。
- 左侧页面树采用平铺数据结构加 `parentId` 的方式实现，避免过早把本地数据改成深层嵌套数组。
