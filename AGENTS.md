# AGENTS.md

## 项目背景

项目名称：`wolai_mvp`

这是一个参考 wolai / Notion 交互形态的个人笔记 MVP，但当前目标不是复刻成熟产品，而是围绕第一版最小闭环持续演进。

第一版最小闭环：

- 用户进入应用
- 创建一个页面
- 输入标题和正文
- 保存内容
- 回到页面列表
- 再次打开刚才的页面
- 能继续编辑

当前产品定位：

- 单用户个人笔记工作台
- 左侧页面列表，右侧编辑区
- 数据暂存在浏览器 `localStorage`
- 优先验证创建、编辑、保存、再次打开，而不是完整知识库能力

## 技术栈

- Vue 3
- Vite
- 原生 CSS
- `localStorage` 持久化

开发命令：

```bash
npm install
npm run dev
```

构建验证：

```bash
npm run build
```

## 当前代码结构

- `src/App.vue`：组合左侧页面列表和右侧编辑器
- `src/components/Sidebar.vue`：页面列表、新建页面、右键删除页面
- `src/components/NoteEditor.vue`：页面标题、块编辑器和页面更新逻辑
- `src/components/BlockEditor.vue`：块级输入、键盘交互、斜杠菜单
- `src/composables/useNotes.js`：页面数据、当前页面、创建、选择、删除、保存
- `src/utils/formatTime.js`：时间格式化
- `src/styles/base.css`：全局布局和编辑器样式

## 已完成功能

页面级能力：

- 页面列表
- 新建页面
- 选中页面高亮
- 页面标题编辑
- 自动保存
- 刷新后恢复数据
- 右键页面打开菜单
- 删除页面

块编辑阶段 1：

- 页面正文由 `blocks` 组成
- 旧 `content` 会迁移为 `blocks`
- 普通文本块编辑
- 回车新增下一个块
- 空块按 Backspace 删除当前块
- 自动保存块内容

块编辑阶段 2：

- `paragraph` 普通正文块
- `heading1` 一级标题块
- `heading2` 二级标题块
- `bullet` 无序列表块
- `numbered` 有序列表块
- `todo` 待办块
- `code` 代码块
- `# ` 转一级标题
- `## ` 转二级标题
- `- ` 转无序列表
- `1. ` 转有序列表
- `[] ` 或 `[ ] ` 转待办块
- 三反引号转代码块
- 待办块可勾选并保存状态
- 多行文本粘贴会拆分为多个块
- 粘贴内容会识别标题、列表、待办和三反引号代码块

块编辑阶段 3：

- 块左侧支持选择入口
- 块左侧支持操作菜单
- 操作菜单支持嵌套的“转换为”子菜单
- 操作菜单支持上移、下移、复制、删除
- 支持多选块
- 支持批量删除选中块
- 删除所有选中块时会保留一个空正文块

块编辑阶段 4：

- 输入 `/` 打开块类型菜单
- 菜单支持鼠标点击选择
- 菜单支持上下键选择
- `Enter` 确认
- `Esc` 关闭
- 选择后转换当前块类型并保持焦点

## 当前数据结构

页面结构：

```js
{
  id: string,
  title: string,
  content: string,
  blocks: Block[],
  createdAt: number,
  updatedAt: number
}
```

块结构：

```js
{
  id: string,
  type: "paragraph" | "heading1" | "heading2" | "bullet" | "numbered" | "todo" | "code",
  text: string,
  checked: boolean
}
```

说明：

- `blocks` 是当前正文编辑的主数据。
- `content` 目前保留作兼容字段，由 blocks 文本拼接生成。
- 不要在当前阶段引入数据库、后端或账号系统。

## 重要历史决策

- 曾接入过 `@toast-ui/editor` 做 Markdown 实时编辑，后来按用户要求回退。
- 当前方向改为自建最小块级编辑体验，而不是 Markdown 编辑器。
- 阶段 3 曾暂时跳过，后来已补齐块操作菜单、移动、复制、删除和批量删除能力。

## 开发约定

- 每个阶段或独立功能完成后提交 git。
- 提交记录使用中文。
- 优先保持最小闭环，不要主动扩展成熟产品功能。
- 修改功能后至少运行：

```bash
npm run build
```

- 如果涉及交互，优先在浏览器中验证关键路径。
